# MVP 需求说明

## 1. 项目目标

为绿色能量与无线射频混合供能网络提供统一的仿真实验管理能力，使用户能够配置网络场景和算法参数、提交实验任务、跟踪执行状态并查看结果。

## 2. 第一阶段范围

第一阶段只完成 Java 后端闭环：

- 创建、查询和修改仿真场景；
- 在提交任务时选择算法并填写训练参数；
- 基于场景、算法和训练参数创建实验任务；
- 校验任务状态流转；
- 保存和查询实验结果；
- 提供统一响应、参数校验和异常处理；
- 为核心业务编写自动化测试。

第一阶段使用 Java 模拟任务执行，不接入 Python，不接入 RabbitMQ。

## 3. 暂不纳入 MVP

- 微服务和服务注册发现；
- Kubernetes；
- 复杂前端页面；
- 多租户；
- 大文件分布式存储；
- 多算法节点动态扩缩容。

## 4. 用户故事

### 场景配置

作为实验用户，我可以保存设备数量、时隙数量、电池容量、数据缓存容量、绿色能量均值和数据包到达率等参数，以便重复使用同一场景。

### 算法配置

作为实验用户，我可以在提交任务时选择算法并保存主要训练参数。第一版不单独维护可复用的算法配置实体。

### 任务提交

作为实验用户，我可以选择一个场景、算法和训练参数创建任务，并看到任务从`PENDING`开始的当前状态。

### 结果查询

作为实验用户，我可以查看长期吞吐量、训练耗时和其他实验指标。

## 5. 完成标准

- 应用可以使用 Java 17 启动；
- 健康检查与基础接口可访问；
- 核心状态规则有单元测试；
- 后续每个业务模块均有明确 API 和测试；
- README 中的命令可以在本机复现。
