## 2. 软件开发岗版

### 2.1 标准版：推荐用于Java后端、软件研发和金融科技岗位

**无线通信仿真实验管理与异步任务调度平台｜个人项目·后端开发｜2026.07—至今**  
技术栈：Java 17、Spring Boot、Spring Security、MyBatis、MySQL、RabbitMQ、Redis、Flyway、Docker、Python/PyTorch

- 面向无线通信仿真实验构建模块化单体后端，实现JWT认证、用户资源隔离、场景CRUD、乐观锁、任务状态机、不可变参数快照及结果查询，使用Flyway管理6张业务表的版本迁移。
- 使用`Idempotency-Key`、规范化请求摘要与数据库唯一约束实现任务提交幂等；在同一MySQL事务中写入任务与Outbox事件，避免业务提交成功但任务消息丢失。
- 基于RabbitMQ实现异步调度，结合Publisher Confirm/Return、发送租约、指数退避、手动ACK、延迟重试和最终死信，使用状态条件、`attemptNo`及唯一约束吸收重复消息的业务影响。
- 使用Redis Cache Aside缓存任务详情，并在事务提交后失效缓存；通过Lua脚本实现用户维度原子限流，Redis异常时回源MySQL或降级放行。
- 接入Python/PyTorch预训练GRPO Worker，完成3设备RSMA吞吐量推理与Java事务落库闭环；构建97项Java测试和10项Python测试，并在真实MySQL、Redis、RabbitMQ及CUDA环境完成全链路验证。

### 2.2 精简版：项目空间只能容纳3条时使用

**无线通信仿真实验管理与异步任务调度平台｜个人项目·后端开发｜2026.07—至今**  
技术栈：Spring Boot、Spring Security、MyBatis、MySQL、RabbitMQ、Redis、Python/PyTorch

- 实现JWT认证、场景管理、任务状态机、参数快照和结果查询；使用幂等键、请求摘要及数据库唯一约束防止重复提交。

| 对比项 | 软件开发岗版 | 央国企科技岗版 |

| 首要目标 | 展示Java后端深度和可靠消息能力 | 展示科研工程化和完整信息系统能力 |

| 核心难点 | 事务边界、重复消息、缓存一致性 | 系统集成、数据可靠、算法落地、可维护性 |

| 测试证据 | 强调测试覆盖和真实组件 | 强调系统验收、文档和可追溯性 |

两版不能互相矛盾。无论投递哪个版本，都应保持以下事实一致：

- Java 97项、Python 10项测试；
- 真实模型只支持3设备RSMA吞吐量评估；
- Python加载预训练GRPO权重，不在线训练；
- MySQL是最终事实来源；
- 消息至少一次投递，业务结果通过幂等规则保持唯一。
